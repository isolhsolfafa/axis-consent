# AXIS 교육 진행 스크립트 (v0.2)

> 대상: 협력사 작업자 (MECH 오전 회차 / ELEC 오후 회차) · 강당 1시간 단발 교육
> 형식: 강사 시연 위주 + 라이브 매뉴얼 사이트 띄움 + 보조 슬라이드
> 마지막 갱신: 2026-05-28

---

## 진행 흐름 한눈에

| 시간 | 구간 | 형식 | 비고 |
|---|---|---|---|
| 0~7분 | **Why DX (정성껏)** | 슬라이드 4장 | 앞단 메시지가 가장 중요 |
| 7~12분 | 큰 그림 | 슬라이드 3장 | OPS·VIEW 관계 + 본인 회차 안내 |
| 12~20분 | **공통 사용법 라이브 시연** | 매뉴얼 사이트 | login → qr-scan → task-workflow |
| 20~38분 | **카테고리별 라이브 시연** | 매뉴얼 사이트 | 회차 분기 (MECH or ELEC) |

| 55~60분 | Q&A + 가입 안내 | 직접 | 동의서 미제출자 처리 안내 |

### [슬라이드 2 — 작업자 입장 이득] (2분)

> 솔직히 말씀드리면, 회사가 새 시스템 도입한다고 하면 작업자분들 입장에서는 **'또 일거리 늘었네'** 하는 생각 드시는 게 자연스럽습니다. 충분히 그러실 만하죠.
>
> 그런데 이번 시스템은 좀 다릅니다. 이게 도입되면 여러분 입장에서 이런 게 바뀝니다.
>
> 첫째, **종이 작업지시서·체크리스트가 사라집니다.** 핸드폰으로 QR 한 번 찍으면 그날 본인이 할 작업이 자동으로 뜹니다.
>
> 둘째, **작업 시작·완료 시각을 본인이 적을 필요가 없습니다.** 버튼 한 번이면 그 시각이 자동으로 기록됩니다.
>
> 셋째, **휴게시간에는 작업이 자동으로 일시정지됩니다.** 점심·휴식 시간만큼 작업 시간 손해 안 보십니다. 이건 사실 작업자분들 직접 요청해서 들어간 기능이에요.
>
> 넷째, **어디까지 했는지 본인 핸드폰에서 바로 확인됩니다.** 다음에 누구한테 인계할지, 본인이 다시 와서 어디부터 할지 한눈에 보입니다.

**[강사 보조]**
- "또 일거리 늘었네" 부분 — 청중 시선 살피며 가볍게 (공감대 형성)

### [슬라이드 3 — GST(원청) 입장 이득] (2분)

> 회사 입장에서도 솔직히 말씀드리면 이런 이유로 합니다.
>
> 지금까지는 **'협력사가 오늘 어디까지 작업했는지'** 매니저분들이 매번 전화·카톡으로 확인해야 했습니다. 이게 안 끝난 사유로 GST에서 협력사로 다시 묻고, 협력사에서 작업자에게 묻고, 다시 답이 올라오고… 이런 과정이 매일 반복됐어요.
>
> 이번 시스템 들어가면 **GST는 작업 진행 상황을 데이터로 봅니다.** 매번 안 물어봐도 돼요.
>
> 그러면 GST는 그 데이터를 가지고 **협력사에 더 정확한 일정을 제공할 수 있게 됩니다.**
> "다음 주 화요일에 ELEC 작업 시작하실 수 있도록 MECH 도킹 일정 맞추겠습니다" — 이런 식으로 일정 협의가 훨씬 명확해집니다.
>
> 결국 **협력사도 일정 예측이 쉬워지고, 작업자분들도 일정이 갑자기 바뀌는 일이 줄어듭니다.**

**[강사 보조]**
- "협력사도 이득 본다"는 점 강조 (단방향 메시지 아니라 양방향)
- GST 입장 너무 깊게 안 들어감 (작업자가 듣고 싶은 메시지 아님)

### [라이브: operator/login-and-permission] (3분)

**보여줄 화면:** 매뉴얼의 "로그인·권한" 페이지

- 승인되면 PIN 4자리 설정 → 다음부터 PIN으로 빠르게 로그인

> **[강조]** 회사·역할 선택은 가입할 때만 본인이 할 수 있어요. 잘못 선택하면 나중에 본인이 못 바꿉니다. **가입 전에 협력사 매니저한테 본인 소속 코드 확인 받으시는 게 안전합니다.**

### [라이브: operator/qr-scan] (3분)

**보여줄 화면:** 매뉴얼의 "QR 스캔" 페이지

**핵심 포인트:**
- 홈에서 "QR 스캔" 메뉴 → 카메라 권한 허용
- `DOC_` 시작은 제품 QR · `LOC_` 시작은 위치 QR — 앱이 자동 인식
- 명판 QR이 작아서 인식 안 되는 경우 → **5~10cm 거리에서 천천히 접근**
- 끝까지 안 되면 기본 카메라 앱으로 텍스트 읽어서 "QR 직접 입력"

> **[강조]** 카메라 권한 팝업이 화면에 가려서 안 보이는 경우가 있어요. 그땐 브라우저 주소창 옆 🔒 아이콘 눌러서 직접 허용으로 바꿔주시면 됩니다.

### [라이브: operator/task-workflow] (2분)

**보여줄 화면:** "Task 작업 흐름" 페이지

**핵심 포인트:**
- 스캔 성공하면 → 제품 정보 + 본인이 할 Task 목록
- **"시작" 버튼 → 그 시각이 자동 기록**
- 진행 중 → 경과 시간 본인 화면에 보임
- 완료 시 **"완료" 버튼** → 다음 Task 자동 추천

**멀티 작업자 (현장에서 흔함):**
- 동료 여러 명이 같은 Task에 붙어서 작업할 수 있음
- 각자 본인 "시작" 버튼 누름
- 본인 작업 끝나면 **"내 작업 종료"** (Task 자체는 열린 상태 유지)
- 마지막 사람이 **"작업 완료"** 누르거나, 모두 본인 종료하면 자동 마감

> **[강조]** 핵심 두 버튼만 외우시면 됩니다 — **"시작"** 과 **"완료"**. 종이에 시간 적던 일이 이 두 번의 탭으로 끝납니다.

## 20~38분 — 카테고리별 라이브 시연 (회차 분기)

### 🌅 오전 회차 — MECH 협력사

**[전환 멘트]**
> 자, 오전 회차는 MECH 분들이라 MECH 특화 부분 보겠습니다. 두 가지가 중요해요.
> 첫째는 **Tank Docking 알림이 어떻게 자동으로 나가는지**, 둘째는 이번에 새로 도입된 **MECH 체크리스트**입니다.

#### [라이브: operator/task-workflow의 알림 영역 + alerts] (3분)

**핵심 포인트:**
- Tank Docking 완료 누르시면 → **ELEC 매니저에게 자동 알림**
- "도킹 끝났습니다, ELEC 작업 시작 가능합니다" 메시지가 알아서 갑니다
- 지금처럼 전화·카톡으로 따로 알릴 필요 없음
- TM(가압검사) 완료도 마찬가지 — MECH 관리자에게 자동 통보

> **[강조]** 본인 입장에서는 그냥 평소처럼 작업 끝나면 **"완료"** 만 누르시면 됩니다. 알림이 누구에게 어떻게 가는지는 시스템이 알아서 처리해요.

#### [라이브: operator/task-workflow의 Worker Pause/Resume] (2분)

**전환 멘트:**
> MECH 체크리스트 들어가기 전에 한 가지만 더 — 작업 중에 잠깐 멈춰야 할 때 쓰는 기능입니다. 5월 업데이트(Sprint 55)로 들어왔어요.

**핵심 포인트 — Worker별 일시정지/재개 (Sprint 55):**

- **개인별 pause/resume** — 기존(Sprint 14~15) Task 단위 pause와 달리, **본인만 멈춥니다**
  - A·B·C 동시 작업 중 A가 일시정지 → A만 멈추고 B·C는 계속 진행
- **일시정지 절차**
  1. Task 카드 → **"일시정지"** 버튼
  2. 사유 선택 (선택사항: 휴식 / 자재 대기 / 기타)
  3. 본인 작업 시간 카운트 멈춤
- **재개 절차**
  1. 작업 관리 화면에서 일시정지된 Task 표시
  2. **"재개"** 버튼 클릭
  3. 작업 시간 카운트 재시작
  4. duration 자동 차감

> **[강조]** 일시정지 시간은 `duration_minutes` 에서 자동 차감됩니다. CT(Cycle Time) 분석에 **정확한 작업 시간만 반영**되니, 휴식·자재 대기 등으로 잠깐 멈추실 일 있으면 부담 없이 일시정지 누르셔도 됩니다. 본인 실적이 부풀려지지도 깎이지도 않아요.

#### [라이브: operator/checklist §4 MECH 체크리스트] (15분)

**전환 멘트:**
> 자, 이게 오늘 오전 회차의 가장 큰 변화입니다. **5월에 도입된 MECH 체크리스트**예요.

**핵심 포인트 (페이지 짚어가며):**
1. **73항목 / 20그룹** — 종이로 하시던 그 체크 항목 그대로 디지털로 옮긴 거예요.
   그룹 — 3Way V/V · WASTE GAS · INLET · BURNER · REACTOR · GN₂ · LNG · O₂ · CDA · BCW · PCW-S · PCW-R · Exhaust · TANK · PU · 설비 상부 · 설비 전면부 · H/J · Quenching · 눈관리.

2. **진입 시점은 4가지 Task** — 각 Task 시작 시 자동으로 토스트 뜸
   - **UTIL_LINE_1** 시작 → Speed Controller 방향·수량 입력
   - **UTIL_LINE_2** 시작 → MFC Spec·Flow Sensor Spec 선택
   - **WASTE_GAS_LINE_2** 시작 → INLET S/N 8개 입력
   - **SELF_INSPECTION (자주검사)** 시작 → 일반 CHECK 54항목 일괄 검수

3. **입력 형식 3가지**
   - **CHECK**: ✅/❌ 토글 (방향 등)
   - **INPUT**: 텍스트 입력 (INLET S/N · 수량)
   - **SELECT**: 드롭다운 (MFC·Flow Sensor 종류 — 실제 자재 데이터 옵션)

4. **INLET S/N 8개 분리** — Left #1~#4 + Right #1~#4
   - 일반 모델은 단순 4개
   - **DRAGON 모델만** L/R 분리 8개

5. **1차 → 2차 단계**
   - 1차는 작업자가 입력 (UTIL_LINE 등 Task 진행 중)
   - 2차는 관리자·QI가 자주검사 시점에 검수 → **1차에 입력하신 값은 자동으로 떠서 read-only로 보입니다**

6. **자재 마스터 연동** (5월 업데이트)
   - MFC·Flow Sensor 드롭다운은 실제 등록된 자재명 (LNG·CDA·O₂·N₂ 4 가스종류 + Maker별 14개 사양)
   - **앱 버전이 v2.12.2 미만이면** 옵션이 임시 텍스트(`"MKS GE50A | 5 SLM | ..."`)로 보일 수 있음 → 앱 업데이트로 해결

> **[강조]** 앱 버전 확인하시고, 구버전이면 업데이트 부탁드립니다.
> v2.11.2 미만 = 진입 버튼 안 보임 / v2.11.5 미만 = 2차에 1차 값 안 떠오름 / v2.12.2 미만 = 자재 옵션이 임시 텍스트.
> **모두 앱 업데이트 한 번이면 해결되니, 가입 후 첫 작업 들어가시기 전에 버전 확인 부탁드립니다.**

#### [라이브: operator/task-workflow의 Worker Pause/Resume] (2분)

**전환 멘트:**
> ELEC 체크리스트 들어가기 전에 한 가지만 더 — 작업 중에 잠깐 멈춰야 할 때 쓰는 기능입니다. 5월 업데이트(Sprint 55)로 들어왔어요.

**핵심 포인트 — Worker별 일시정지/재개 (Sprint 55):**

- **개인별 pause/resume** — 기존(Sprint 14~15) Task 단위 pause와 달리, **본인만 멈춥니다**
  - A·B·C 동시 작업 중 A가 일시정지 → A만 멈추고 B·C는 계속 진행
- **일시정지 절차**
  1. Task 카드 → **"일시정지"** 버튼
  2. 사유 선택 (선택사항: 휴식 / 자재 대기 / 기타)
  3. 본인 작업 시간 카운트 멈춤
- **재개 절차**
  1. 작업 관리 화면에서 일시정지된 Task 표시
  2. **"재개"** 버튼 클릭
  3. 작업 시간 카운트 재시작
  4. duration 자동 차감

> **[강조]** 일시정지 시간은 `duration_minutes` 에서 자동 차감됩니다. CT(Cycle Time) 분석에 **정확한 작업 시간만 반영**되니, 휴식·자재 대기 등으로 잠깐 멈추실 일 있으면 부담 없이 일시정지 누르셔도 됩니다. 본인 실적이 부풀려지지도 깎이지도 않아요.

#### [라이브: operator/checklist §3 ELEC 체크리스트] (15분)

**핵심 포인트:**
1. **Phase 1 + Phase 2 분리**
   - Phase 1: 1차 배선 — I.F 1 완료 후, 약 15항목
   - Phase 2: 2차 배선 — I.F 2 완료 후, 약 16항목
   - 합계 31항목

2. **항목 형식 3종** (MECH 회차와 동일 구조)
   - CHECK · INPUT (측정값 단위 포함) · SELECT (TUBE 색상 등)

3. **검사자 분기 — JIG WORKER vs QI**
   - JIG WORKER: 본인이 체크 (회색 배지)
   - QI: GST 검사원이 체크 (보라색 배지)

> **[강조]** QI 항목은 본인이 체크해도 실적 판정에 반영 안 됩니다. 색깔로 구분되니까 본인 항목만 챙기시면 됩니다.

4. **INSPECTION freeroll**
   - ELEC 자주검사는 phase 무관하게 언제든 진입 가능
   - I.F 2의 FINAL 처리(작업 완료) 트리거

5. **DUAL 모델 (L/R 분리)** — GAIA·GALLANT 등 2-Tank 모델
   - 좌측: qr_doc_id 끝이 `-L`
   - 우측: qr_doc_id 끝이 `-R`
   - **각각 별도 체크리스트 + 별도 제출**

> **[강조]** DUAL 모델은 한쪽만 완료하시고 다른 쪽 빠뜨리는 실수가 자주 나옵니다. **L 끝나면 무조건 R도 확인** — 같은 작업 두 번 한다는 느낌으로 가시는 게 안전합니다.
